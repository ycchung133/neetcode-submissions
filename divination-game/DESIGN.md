# 演算法籤詩 — 關卡與敘事設計

## 主題
玩家走進「演算之廟」，每座神殿供奉一道 NeetCode 題。要求神明指點，得先**擲筊**問卜、
或以**吊酒瓶**的巧勁擺正資料結構；求得的「籤」由 **AI 解籤師**（重用你的解籤 LLM）
詮釋成漸進提示。三聖筊（過三關）即修復神殿。

## 兩套既有物理各自的用法

### 擲茭引擎 → 「決策 / 隨機 / Heap」機制
- **落地 3 態**本身就是離散狀態機的天然範例，完美示範計畫書的核心原則。
- 用於需要「做出一次決定並 commit」的題型，或用擲筊的隨機性生成題目測資。
- 進階：Heap 呈現——擲出的筊依「數值密度」浮沉，最大值上浮（對應計畫書的浮力水槽）。

### 吊酒瓶引擎 → 「Linked List / Tree / 平衡」機制
- 繩索=指標、節點=瓶/球、鉤子=玩家操作。
- **Reverse Linked List**：實際 `cut` 繩再 `link` 回反向。
- **Invert Binary Tree**：吊飾左右鏡射並靠平衡物理維持不倒。
- **吊酒瓶精準度**天然對應「一步錯全盤晃」的演算法直覺。

## 解籤師（AI 導師）
- 玩家「求籤」= 呼叫 `OracleInterpreter.interpret()`。
- `revealLevel` 對應求籤次數：第一次給玄一點的方向，多求幾次才貼近解法——把「不直接給答案」包裝成解籤的儀式感。
- `highlight` 讓解籤師「指點迷津」，在場上點亮某個節點。

## 首個垂直切片（新 folder 的 M0）

且 `cut`/`link` 手勢直觀。解籤師用你的 LLM 給「先想想尾巴要變頭」這類漸進提示。
